Ошибка в именах анимаций.
 
1. Либо указано что-то вовсе не ведомое игре Idles: Start (например)
2. Либо - только что-то локальное Sound: Moh Muh
3. Либо - тоже что-то локальное: SoundGen: Moan
НО! Когда был открыт редактор без подключения мастер файлов.
 
1. Т.е. игра может использовать только жестко зашитый список групп анимации.
Любое неизвестное название приведет к сообщение об ошибке. Idles: Start - неправильно! Idle: Start - правильно!
В этом случае, проверьте указанные названия в Кф файле.
 
2. Т.е. игра не может найти звуковой файл указанный для проигрывания.
Здесь следует учитывать, что ID звука должен быть правильно указан и он должен быть прописан в разделе звуков плагина.
Также, недопустимо использовать пробелы в названиях.
Проверьте ID звука и если есть пробел в нем, замените на _ (нижнее подчеркивание), затем поправьте Кф файл.
 
3. Т.е. ошибка о проблемах анимации может возникать при открытии пустого редактора, без активации мастер файлов.
И подключении модели в раздел Creatures. В этом случае, ошибку следует полностью игнорировать.
Т.е. такое обычно делается в целях быстрого теста сеток, а не игровых возможностей существа.

Right Eval, Left Eval, Infix to Postfix
http://mwmodders.com/scriptingnotes.html (С)
 
Certainly the most dreaded errors one will encounter in scripting are 'Right Eval,' 'Left Eval', and 'Infix to Postfix.' These errors do not show in the compiler, rather, they will only spring up when the dirty line of code causing them fires. Which means, in many cases, one may be able to successfully test much of a script until a certain area fires. In his guide, GhanBuriGhan cites specific causes for these errors that may cause them. I have always found, however, that differentiating between the three errors is virtually useless, as any dirty line can seemingly cause any of the three dreaded errors. Here are some tips if you find yourself in a predicament with one of these errors:
 
    34th variable - make sure you NEVER use the 34th variable declared in the script. Shorts and floats count separately, so it's helpful to keep the short declaration section separate from the float declaration section, so that counting each is easier. Declare the 34th variable as "DontUse" or something similar, to remind yourself not to use the variable. Attempting to set a 34th variable to any value will almost certainly yield an error. Usually a Right Eval.
    Undeclared Variable - when you're dealing with a lot of different scripts and use similar variable names among them, especially temporary variables, it's easy to forget to declare a variable, and luckily, the compiler will usually catch it. However, if the variable has been declared in ANY other script in the game, the compiler will treat it as declared and will not give a compile error, even though the variable hasn't actually been declared in the script. This will usually yield a Right Eval.
 
Together, these two make up about 90% of the evals that I see occurring in my own scripts.

Greatness7  писал:
It's a memory cleanup thing.
When the executable is terminated it checks if any texture references are still around in memory when they (probably) shouldn't be and reports the reference counts.
Might be worth looking into by NullCascade if you've found some nif setup that always leaks these references.

 #INFO or #INDO in pick root.
 
 #INFO or #INDO in pick root. Show the scene graph to find the bad bound radius under world pick root.
 
Проблема с просчетом радиуса у активируемого объекта.
Т.е. какой-то шейп в составе ниф файла приводит игру в "ступор" не позволяя правильно рассчитать на какой дистанции показывать игроку сообщение с названием объекта.
Может возникать при телепортации в ячейку "мертвых" нпс, в т.ч.
Т.е. если НПС был убит в одной ячейке и игрок это "видел", а затем, "трупик" был перемещен к игроку в новую ячейку.
А равно с системами частиц которые были созданы через нифскоп, но не получили значение в разделе радиус.
 
Вероятны и другие случаи.
 
Можно найти проблемный ниф файл в редакторе по Ф10, либо в игре по SSG.
Он помечается #nif (-#ind, #ххх) добавленной к строке WorldBound.
WorldBound - да, это название радиуса объекта.
 
Лечение.
- Указать некоторый радиус проблемному шейпу.
- Добавить ББ, это будет также влиять на просчет дистанции активации и, возможно решит проблему без правки радиуса.
Но лучше совместить. Поправив радиус и добавив ББ.
- Не телепортировать к игроку "мертвых" НПС или сущест.
